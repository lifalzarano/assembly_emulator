Using this site is easy!

To run assembly code, type your code in the box and press 'Run!'. Any results will appear in the console in the middle of the page. This console can be moved or resized, by the way - just drag it around or pull on the edges.


Syntax

Commands do, however, need to have specific syntax. Everything must be separated by spaces:

LDR R0, =42

is valid, while

LDR R0,=42
ADDR0, R1,R2

are not.


Proper spacing is crucial in push and pop statements in particular. The brackets that enclose the list of registers to be pushed/popped must be separated by spaces from said registers:

PUSH { R0, R1, R2 }
POP { R0, R1, R2 }

Comments

To comment out a line, place a semi-colon at the very front:

;MUL R0, R1, R2

Loading Values From C Code

It is possible to specify variables of various sizes in C, and then use these variables in Assembly. The following types can be used:

// Spaces aren't too particular in the C code section.
int x = 2;
// This is equally valid as above - spaces around the equal sign don't matter.
int x=17;

// Only stores 16 bits - sign extends to 32 bits
short s1 = 12;
short int s2 = 21;

// Load a regular 32-bit value
int x = 42;
long y = -42;
long int z = 42;

// Load a 64-bit value
long long x = 900000000000;
long long int y = 200000000000;

// Signed or unsigned can be specified before any data type
unsigned int u32 = 25;
signed long s64 = -12;

*Note: specifying signed or unsigned won't make a difference. The binary value of the number specified is what gets loaded, so it doesn't have any effect.


To load a C-specified value into a register, type anything like the following into the Assembly Code section:

LDR R0, u32
LDRH R1, u16
LDRSH R2, s16

For a full reference on how each variation of LDR works, see the 'Command Reference' section.
